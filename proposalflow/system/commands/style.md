# /proposal style

## Stage 3: Style Matching

## Arguments
- `$ARGUMENTS`: Path to the proposal folder

## Process
1. Read `system/agents/03-style-matcher.md`
2. Verify `client_brief.md` exists (prerequisite)
3. Load style guide from `system/style/`
4. Check for EngageOps for example proposals; fall back to `system/examples/`
5. Analyze 2-3 relevant examples
6. Produce `style_profile.md`
7. Update `metadata.json`, suggest `/proposal interview <folder>`
