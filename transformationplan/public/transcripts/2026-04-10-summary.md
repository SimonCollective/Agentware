- Edge types with named relationships: account manager, consultant, owns, participates in, proposal for, etc.
- Rob demos the HTML visualiser for Simon and Kate's graph.
- Corrections made live during the session:
  - Location updated to **Nottingham**.
  - Simon's business unit renamed to **Collective Security** (from "Collective Intelligence").
  - Proposal, engagement, and document node types added.
- Alexandria sits alongside a separate temporal memory layer ("what Simon did this week"). Alexandria is for structural facts; the other layer is for history.
- Both Simon and Kate's AIs share the same Alexandria graph.
- Future plan: nightly jobs that walk HubSpot, score email sentiment, and update the graph automatically — eventually reducing dependency on HubSpot itself.

### Storage Philosophy

- Simon has set up Supabase, but Rob clarifies the graph doesn't need it — plain JSON in GitHub with local copies on a NAS or Raspberry Pi works fine and gives resilience and nightly backup.

### GitHub Repo vs Working Folders

- GitHub repos hold **only the bare software** — flows, templates, graph structure.
- Real client work (proposals, transcripts, data) lives in **local or OneDrive folders**, never committed to GitHub.
- Kate doesn't need to touch GitHub at all; Simon and Rob use it only for versioning code and templates.

- Simon clones the Agentware repo into his Cowork environment.
